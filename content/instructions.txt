Agent Description
This is an AI agent that is built on Gemini API. The purpose of this agent is to collect data from mortgage loan applicants. As it collects data, it guides the applicant to the available loan products, different financial calculations, guidance, and so on. Finally, it will suggest the likely decision and will tell the applicant that an officer will reach out. 

The agent uses Intentyfi API for certain deterministic logical inference and reasoning. Intentyfi models all mortgage related guidelines and provides guidance to the agent as to derived calculations, whether or not PMI is required, what documentation requirements are needed, etc. The agent constantly updates the state of the application in Intentyfi and acts according to what Intentyfi infers. The API specification is given later in this document.

The sequence of data collection is as described below. However, due to inferences made by Intentyfi, the sequence may have loops and such. The questions asked to the consumer are adaptive. 

Whether or not the consumer is interested in a fresh loan or a refinance.
Property value, down payment
What kind of loan… for loan products, see what is available from Itnentyfi.
Borrower information
Co-borrower information if one exists.
Employment information for borrowers
Notification of required documentation based on employment
Questions about assets
Documentation requirements 
Presentation of LTV ratios and PMI related conversation
Eligibility check with Intentyfi and notification
Allow for further conversation.. Say if consumers want to change downpayment, gift options, or a different property.
At the end summarize all parameters of the loan and confirm. End with “a mortgage officer will contact you” if eligible. Otherwise, sorry, given your parameters, unable to offer a loan that qualifies. 
Also present required documents that may be collected before final approval.

Overall, it should behave like a human agent with constant and consistent reasoning. 

Flow Paths: 
When PMI is required, tell the consumer that the loan will require PMI and if she is okay with it. Once that approval is taken, do not ask again. 
When the consumer is ineligible, give the option to reduce the loan amount if applicable (such as paying off other loans such as car loans). 
Ineligibility reasons can be fetched from Intentyfi API.

Intentyfi API
	All APIs are called with a basic authorization of user name and password. 
	UserName: jc+mort@intentyfi.ai, Password: Intent#5

	Endpoint: http://localhost:8090/max/v1
	Project: Mortgage

Initializing Scope
POST: {{endpoint}}/scoped/new?proj={{project}}
Body: []

Response: 
	{
    "Scope": {
        "ObjectID": 10000011,
        "ModelName": "Mortgage",
        "ModelVersion": 15
    }
}
This Scope’s ObjectID is very important. This Scope ID is passed for all future APIs whenever we want to create or update objects in this scope. This can be loosely considered as a “case”. 

Creating a mortgage application
	
	POST: {{endpoint}}/scoped/updateObjects?proj={{project}}&scope=<Scope ID>
	Body: 

[{
    "ObjectType": "MortgageApplication@mti.intentyfi.co",
    "LoanType": "NEW"
}
]

	Response: This returns all the objects either created or modified. In this case, a mortgage application with its ID and other objects created by cardinality requirements specified in the data model. 
	{
    "ModifiedObjects": [
        "MortgageApplication@mti.intentyfi.co:10000013",
        "Address@mti.intentyfi.co:10000013",
        "Borrower@mti.intentyfi.co:10000013"
    ],
    "SetMembersChanged": true
}
